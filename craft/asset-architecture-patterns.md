## Email sequence shapes

Default architectures for common sequence types. Each is a starting point; the Brief + Concept may justify divergence (flag the override with reasoning in the Agent Plan).

### Welcome sequence (default: 5 emails)

| # | Role | Default timing | Concept-beat lens | CTA escalation |
|---|---|---|---|---|
| 1 | Deliver / confirm + welcome | Trigger + 0 (immediate) | Brand introduction | Low-friction — read / click |
| 2 | Quick win + value drop | +1 day | Establish utility | Low — engage with content |
| 3 | Story / credibility / "why we exist" | +3 days | Brand narrative | Mid — follow / save |
| 4 | Introduce core offer gently | +5 days | Product-value bridge | Mid — explore product |
| 5 | Stronger ask + social proof | +7 days | Conversion moment | High — book / try / buy |

### Nurture sequence (default: 4–8 emails, often ongoing)

Pattern: alternate value-giving with soft CTAs. **Never sell in more than 40% of nurture emails.**

Subject-angle variation across the sequence is essential — don't ship 5 curiosity-led subjects in a row.

### Launch sequence (default: 5–7 emails)

| # | Role | Default timing | Concept-beat lens |
|---|---|---|---|
| 1 | Anticipation tease | -7 days | "Something is coming" |
| 2 | Reveal | -3 days | The big idea unveiled |
| 3 | Build the case | -1 day | Proof + handle objections |
| 4 | Final pre-launch | Launch day morning | Social proof / urgency |
| 5 | Live announcement | Launch hour | The CTA |
| 6 | (Optional) Deadline reminder | +1–2 days post-launch | Last call |
| 7 | (Optional) "You didn't act" wrap | +3 days | Final ask or wrap |

### Re-engagement sequence (default: 3–4 emails)

| # | Role | Default timing |
|---|---|---|
| 1 | "We miss you" / "Are you still interested" | Trigger + 0 |
| 2 | Quick win or value drop | +3 days |
| 3 | Specific reason to come back | +7 days |
| 4 | (Optional) "Should we remove you?" — urgency through potential loss | +14 days |

### Onboarding sequence (behaviour-triggered; default 3–7 emails)

Triggered on signup; subsequent emails fire on user action (or non-action).

- **Activate**: trigger user's first key action
- **Adopt**: trigger user's second key action
- **Expand**: introduce a feature beyond core use
- **At-risk-of-churn**: if usage drops below threshold, intervention email

### Abandoned cart sequence (default: 3 emails)

| # | Role | Default timing |
|---|---|---|
| 1 | "You left something" + cart contents + return link | +1 hour |
| 2 | "Still thinking?" + objection handling (shipping / returns / sizing) | +24 hours |
| 3 | Final reminder; sometimes with small incentive | +72 hours |

### Post-purchase sequence (default: 3–5 emails)

| # | Role | Default timing |
|---|---|---|
| 1 | Receipt + welcome | Immediate |
| 2 | Set-up / how to use | +1 day |
| 3 | Social proof / community | +5 days |
| 4 | Cross-sell or referral ask | +14 days |
| 5 | Review request | +21–30 days |

### Transactional flows (variable)

One email per system event. Plain language, mobile-first, minimal marketing — brand voice still applies. Required types vary by business:

- Password reset
- Payment confirmation / receipt
- Shipping notification
- Account locked / security alert
- Subscription renewal
- Refund processed
- Account closed / data deletion confirmation

## Thread architecture

For X threads and reply-chain Threads posts. Length sweet spot: 5–6 posts; 7 is the ceiling for most B2B content unless substance demands more.

### Default 5–6 post structure

| # | Role | Architecture rule |
|---|---|---|
| 1 | **Hook tweet** | Must stand alone — readable if quote-tweeted with no context |
| 2 | Build the pattern | Specific example or proof |
| 3 | Develop / expand | Deeper into the substance |
| 4 | Objection-handler or reframe | Pre-empt the obvious pushback |
| 5 | Land the point | The payoff |
| 6 | **CTA / link / synthesis** | Link goes here (never in middle posts); standalone-readable if quoted |

### Architecture rules

- **Hook tweet stands alone** — readers quote-tweet single posts; the hook must read complete with zero context
- **No filler posts** — every post earns its place
- **Each post earns the next** — close each post with something that pulls the reader to swipe / read on
- **Link in final post only** — links in middle posts kill thread-completion rate
- **Quotable line per post** — threads are shared via single-post quote; design at least one tweet-worthy line in each post

### Length rationale

Thread length should match substance. 7 posts is a ceiling for most B2B content because completion rate drops sharply after 5–6 posts. Some narrative threads (story arcs) sustain longer; some data threads (chart-per-post) sustain shorter (3–4 posts).

## Carousel architecture

For Instagram carousels (10 slides max), LinkedIn document carousels (12+ allowed), TikTok carousels (10 image max).

### Default 9-slide structure (Instagram / LinkedIn-optimal)

| Slide | Role | Composition rule |
|---|---|---|
| 1 | **Cover** | Hook + visual stop-power; the swipe-or-scroll decision lives here |
| 2 | Setup / problem | Frame what the carousel is about; specific |
| 3 | Pattern / observation | The first piece of substance |
| 4 | Proof / example | Concrete |
| 5 | Pattern continued | Second piece of substance |
| 6 | Reframe | The pivot — the unexpected angle |
| 7 | Payoff | The conclusion the carousel was building to |
| 8 | Recap (optional) | Compress the carousel into one slide for the skimmer |
| 9 | **CTA** | The action — save / follow / read / book |

### Architecture rules

- **Cover slide is the work** — 80% of swipes are decided by the cover
- **Cover visual register differs from interior** — signals "swipe me" (often: bigger type, more colour, more contrast)
- **Each interior slide earns the next swipe** — leave a question, drop a hook, or land a piece that demands the next
- **Final slide CTA unambiguous** — name the action; pair with caption's link reference
- **7–10 slides total** — fewer feels thin; more loses swipe-through

### Slide composition limits

- **3–8 word headlines** max per slide — readable in 2 sec
- **1–3 short lines body** per slide — supporting copy, not paragraphs
- **No paragraphs on slides** — if it doesn't fit in 5 lines max, restructure
- **Visual register varies across slides** — typography-only / chart / illustration / photo / hybrid — don't ship 9 identical-looking slides

## Short-form video architecture

For TikTok / Reels / YouTube Shorts (15–90s; 30–45s sweet spot).

### Default beat structure

| Beat | Duration | Three signals firing |
|---|---|---|
| **Hook** | 0–3 sec | Pattern-interrupt visual + spoken hook + on-screen text — all three at once |
| **Payload** | 3 sec – (N-3) sec | The substance: framework / demo / story / proof |
| **Close** | Last 2–3 sec | Verbal CTA + on-screen text echo + final visual beat |

### Architecture rules

- **First 3 seconds decide >50% of completion** — hook is the work
- **Sound-off friendly** — assume muted; on-screen text must carry the hook (and ideally the whole video)
- **Eyes-elsewhere friendly** — spoken script should make sense even if user is half-looking
- **Cover frame designed** — the thumbnail before tap-to-play; high-contrast moment with on-screen text from hook
- **Verbal CTA in last 2 sec** — caption echoes
- **Speaker delivery notes** in script — pace, energy, pauses for the human performing

### Format-specific variations

- **TikTok**: trend-native; trending sound matters; edit-heavy / cuts-heavy; native ad-coded production polish can backfire
- **Reels (IG)**: same form, less trend-driven, more brand-polished tolerance
- **Shorts (YouTube)**: better for tutorial / education; hooks match search intent; less trend-dependent

## Content calendar mix ratios

Default ratios for a social content calendar across a week or month. Override with Brief / Concept signal.

### Default mix (per platform)

| Content type | % | Purpose |
|---|---|---|
| **Value** (tips, how-tos, frameworks, cheat sheets) | 40% | Earn trust through utility |
| **Opinion / thought leadership** (takes, trends, predictions, contrarian POV) | 25% | Build authority and distinct voice |
| **Engagement** (questions, polls, hot takes, "agree or disagree", AMA-style prompts) | 20% | Drive interaction; algorithm signals |
| **Promotional** (product mentions, CTAs, offers, lead magnets, case studies tied to product) | 15% | Direct revenue |

### Mapping to posting frequency

- **5 posts/week (LinkedIn)**: 2 value + 1 opinion + 1 engagement + 1 promo
- **3 posts/week (LinkedIn lower frequency)**: 1 value + 1 opinion + 1 engagement-or-promo (alternate weeks)
- **5 posts/week (X)**: similar ratios, but engagement posts (replies / quote-tweets / hot takes) blur into "opinion" — adjust ratio toward 40 value / 30 opinion / 20 engagement / 10 promo
- **3 posts/week (Instagram)**: 1 carousel (value) + 1 Reel (opinion or value) + 1 single post (engagement or promo)
- **Daily TikTok**: leans heavier into entertainment / opinion / trend participation — 50% opinion-or-entertainment / 30% value / 20% promo or engagement

### Campaign-phase ramp

- **Phase 1 (build)**: lower frequency, consistent presence, mix heavy on value (testing what resonates)
- **Phase 2 (launch)**: peak frequency; multi-touch per platform per day acceptable; promo % temporarily rises to 30–40% during launch window
- **Phase 3 (sustain)**: dropped frequency, mix returns to default, focus on community engagement and follow-up content

## Repurposing matrix

When a major asset exists (long-form article, presentation script, whitepaper, keynote VO), repurpose for social — typically 3–5 social pieces drawn from one source. Each piece is a *different angle*, not the same idea repackaged.

### Default repurposing chain — long-form article

| Source | Repurposed asset | Angle shift |
|---|---|---|
| 2,000-word article | LinkedIn post (200–400 chars hook + body) | POV teaser — single line that summarises the article's controversial claim |
| Same article | X thread (5–6 posts) | Argument breakdown — each post is one beat of the article's logic |
| Same article | Instagram carousel (9 slides) | Visual reframe — turn the article's data / framework into slide-by-slide visual |
| Same article | TikTok / Reel (45s) | Hook + payoff video — the single most surprising line from the article, dramatised |
| Same article | LinkedIn newsletter excerpt | Newsletter-format teaser with link to full article |

### Default repurposing chain — keynote / presentation

| Source | Repurposed asset | Angle shift |
|---|---|---|
| 15-min keynote | LinkedIn post | Talk takeaway + video clip |
| Same keynote | X thread (5–7 posts) | Talk's key arguments serialised |
| Same keynote | TikTok / Reel (60s) | Best 60 seconds of the talk cut + on-screen text |
| Same keynote | Multiple TikTok cut-downs (3–5) | One per memorable line or beat |
| Same keynote | Static quote cards (5–10) | One quote per card; carousel-ready |
| Same keynote | Blog post / long-form article | Talk transcript reworked as written narrative |

### Default repurposing chain — whitepaper

| Source | Repurposed asset | Angle shift |
|---|---|---|
| 20-page whitepaper | LinkedIn carousel (10 slides) | Distill the whitepaper's frameworks into visual format |
| Same whitepaper | LinkedIn newsletter | 800-word newsletter-format summary with download CTA |
| Same whitepaper | Email nurture sequence (5 emails) | One core insight per email |
| Same whitepaper | X thread (7 posts) | Each post is one finding |
| Same whitepaper | Webinar / presentation | Walk through the whitepaper live |

### Repurposing principles

- **Each adapted piece is a *different angle***, not 5 paraphrases of the same idea
- **Platform-native execution** — each piece obeys its platform's grammar (length, hashtag, hook rules)
- **Source link in every adapted piece** — drive readers back to the long-form
- **Adapted pieces don't replace the source** — they amplify it
- **Cross-piece check** — ensure the 5 adapted pieces feel different from each other

## Landing page anatomy

Default structure for a conversion-focused landing page. Order matters; sections can be combined or split based on offer complexity.

| Section | Role | Length |
|---|---|---|
| **Hero** | Headline + subhead + primary CTA | Headline 8–14 words; subhead 15–30 words |
| **Social proof bar** (logo strip, testimonial quote, rating) | Credibility in first scroll | One line of social proof |
| **The problem** | Frame the reader's pain | 1–2 short paragraphs |
| **The solution** (your product) | Present the product as the reframe | 1–2 paragraphs + visual |
| **How it works** | Reduce friction with clear mechanic | 3 steps OR explainer video |
| **Feature / benefit blocks** | Specific functionality tied to outcome | 3–6 blocks; each 30–60 words |
| **Case study / testimonial** | Proof of outcome | 1–3 testimonials with name + role; case-study with metric |
| **Pricing or offer** | The ask | Pricing table OR clear offer block |
| **Objection-handling FAQ** | Pre-empt the top 5–8 objections | Q&A format; 5–8 questions |
| **Final CTA** | The closing ask | Repeat hero CTA with urgency / specificity |
| **Footer mandatories** | Legal / sitemap / contact | Standard footer |

### Architecture rules

- **One decision** the reader is being asked to make — everything supports it
- **CTA appears 2–4 times** at scroll-strategic points (hero, after social proof, after how-it-works, final CTA, sticky-nav if applicable)
- **Mobile-first** — every section must work in mobile portrait
- **Above-the-fold** earns the scroll — hero + social proof bar
- **AEO discipline** — H1 frames the page's intent; H2/H3 are answerable; first paragraph directly answers the H1's implicit question

## Long-form article structures

Default shapes for articles ≥1,000 words.

### POV / opinion article

| Beat | Role |
|---|---|
| **Hook** | A specific moment or observation that sets up the POV |
| **The conventional wisdom** | Name the thing you're arguing against |
| **The crack in the conventional wisdom** | Where the existing thinking breaks |
| **The reframe** | Your POV — the new way of seeing |
| **The proof** | Specific examples, data, or experience |
| **The implications** | What changes if the reader accepts the POV |
| **Land the point** | The closing thought — not a recap |

### Explainer / educational article

| Beat | Role |
|---|---|
| **Hook** | A specific question or problem the reader is trying to solve |
| **What it is** | Direct definition — front-loaded |
| **Why it matters** | The stakes — why the reader should care |
| **How it works** | The mechanic — step-by-step or principle-by-principle |
| **When to use it** | Application — where this knowledge applies |
| **Common mistakes** | What goes wrong + how to avoid |
| **Next steps** | What the reader does with this knowledge |

### Case study

| Beat | Role |
|---|---|
| **The challenge** | Specific situation + named entity + measurable problem |
| **The context** | Stakes — why the problem mattered |
| **The approach** | What we did — specific decisions and reasoning |
| **The work** | How it played out — choices, course-corrections |
| **The outcome** | Measurable result — named metric |
| **The lessons** | Generalisable insights for the reader's situation |

### Listicle (if the form genuinely fits)

| Beat | Role |
|---|---|
| **Hook + framing** | Why this list matters; what the reader gets |
| **Item 1–N** | Each: short context + specific example + actionable insight |
| **Synthesis** | What the items teach together; not just a recap |

**Listicle anti-pattern**: a list of generic items pretending to be insight. Each item must earn its place.

## Presentation / talk arcs

Default narrative arcs for scripted live performance.

### TEDx-style talk (12–18 min)

| Beat | Duration | Role |
|---|---|---|
| **Open** | 0–60 sec | Hook — a specific moment, story fragment, or surprising claim |
| **Stakes** | 60–180 sec | Why this matters to the audience |
| **Old way** | 3–5 min | The conventional wisdom you'll challenge |
| **The shift** | 5–8 min | The reframe / new way |
| **Evidence** | 8–12 min | Specific proof, story, data |
| **Implications** | 12–15 min | What this changes |
| **Close** | Last 60 sec | Memorable line that crystallises the talk |

### Conference keynote (30–45 min)

Like TEDx but with more time for evidence + implications. Often opens with a personal story, threads through multiple sub-arguments, lands on a strategic call-to-action.

### Webinar (30–60 min)

| Beat | Duration | Role |
|---|---|---|
| **Open** | 0–3 min | Hook + agenda + housekeeping |
| **Frame the problem** | 3–10 min | Why we're here, what we're solving |
| **Main content** | 10–45 min | The substance — usually 3–5 sub-points with examples |
| **Q&A** | 45–55 min | Audience questions |
| **Close + CTA** | Last 5 min | The action — book a demo / sign up / download |

### Town hall / all-hands talk (15–30 min)

| Beat | Role |
|---|---|
| **Open** | Greeting + frame |
| **The update** | Where we are |
| **The forward look** | Where we're going + why |
| **Acknowledge the hard parts** | Don't gloss; name what's tough |
| **Close** | Specific call-to-action or commitment |

### Manifesto VO (60–120 sec)

| Beat | Role |
|---|---|
| **Open** | Hook — a single defining line |
| **Shift** | The pivot — what's changing or what we're calling for |
| **Land** | The closing line — memorable, brand-anthem |

## Display ad sequencing

For retargeting / programmatic display ad sequences where multiple creatives serve to the same user over time.

### Default 3-touch retargeting sequence

| # | Role | Trigger |
|---|---|---|
| 1 | Re-introduce + reason | User visited site; serve within 24h |
| 2 | Objection-handler or social proof | If user didn't convert after touch 1; serve within 72h |
| 3 | Urgency / final CTA | If user didn't convert after touch 2; serve within 7 days |

### Sequencing rules

- **Each creative is distinct** — don't serve the same banner three times
- **Frequency capping** — 3–5 impressions per user per week max (avoid ad fatigue)
- **Drop-off at touch 3** — if user hasn't converted by touch 3, the campaign isn't going to convert them; stop spending
