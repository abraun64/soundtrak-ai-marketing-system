# Platform-native social craft — universal reference

**Read by**: Social/Community Manager (primary), Copywriter–Short-form (for paid social), Creative Director (when authoring Concept §6 social channel hooks), future Static Designer + Motion Designer (for visual + video production).

Each platform has its own reader contract, scroll-and-stop physics, length conventions, ranking signals, community norms, and glance time. **Treat every platform as native by default** — the same post copy-pasted across LinkedIn, X, IG, and TikTok lands on none.

## Platform-by-platform reference

### LinkedIn

**Audience**: B2B professionals; recruiters; founders; execs; aspiring careerists.

**Reader contract**: scrolling at lunch / commute / between meetings. Tolerates longer posts than other social if the hook earns the read.

**Length**: 1,200–1,800 chars ideal; 3,000 max. Posts under 300 chars usually under-perform.

**Hook rules**:
- ≤15 words
- Avoid: "I'm excited to announce" / "I'm thrilled to share" / "It is with great pleasure" / "Quick announcement" / "Big news"
- Best openers: specific number, specific story moment, contrarian POV, direct observation
- Mobile cutoff at ~210 chars before "...more" — the hook lives in those first 210 chars

**Format**:
- Hook line → line break → body (3–5 short paragraphs, 1–3 sentences each) → CTA
- Generous line breaks (every 1–3 sentences) — single-block paragraphs hurt readability
- Bold / italic via Unicode trickery is acceptable for one key phrase per post; overuse flags
- No emoji-bullet runs (✅ ⚡ 🚀) — AI tell

**Hashtags**: 3–5 at the very end of the post, CamelCase for accessibility.

**CTA**:
- Soft CTA in body (read / share / comment-thoughts) outperforms hard CTA
- "Buy now" / "Sign up" in body kills reach — LinkedIn's algorithm downranks outbound links
- Link strategy: link in first comment instead of body (preserves reach)
- Comment-prompt CTAs ("what would you add?") perform well

**Algorithm signals**:
- Comments > likes > shares > views (weighting roughly)
- First-hour engagement determines reach trajectory
- Dwell time matters — posts read in full > posts skimmed
- Outbound links suppress reach; mentions of competitors / political content suppresses

**Sub-form variations**:
- **LinkedIn post (text)**: above
- **LinkedIn carousel** (document upload): 7–12 slides ideal; cover slide hook; final slide CTA; PDF format
- **LinkedIn video**: native upload outperforms YouTube embed; 1–3 min sweet spot; captions mandatory
- **LinkedIn newsletter**: lives in feed AND inbox; behaves like a hybrid email/post; longer than a post, shorter than a Substack
- **LinkedIn poll**: 4 options max; closes after 1 week; engagement bait if used too often

### X / Twitter

**Audience**: B2B + tech + journalists + politicos + crypto + indie hackers + cultural commentators.

**Reader contract**: rapid scroll. Most tweets see < 3 seconds of attention. Quote-tweet culture means every post should stand alone if quoted with no context.

**Length**: 280 chars per post. Threads of 3–7 posts work; longer threads lose readers.

**Hook rules**:
- Hook tweet must stand alone — readable and understandable if quote-tweeted with zero context
- Specific > clever
- Number-led and contrarian-led hooks work especially well
- Avoid generic openers ("Hot take:" / "Unpopular opinion:" — overused AI tells)

**Format**:
- Single tweet: hook → payoff → optional close
- Thread: hook tweet (1/) → numbered posts (2/ 3/ ...) → final tweet with CTA / link / payoff
- "🧵" emoji as thread indicator: optional and slightly dated — use if brand voice supports
- Line breaks within a tweet are fine and often help readability

**Hashtags**: 0–2 inline only. Hashtag spam looks amateur. Often zero hashtags is right.

**CTA**:
- Link in final post of thread, never middle (links in middle kill thread-completion)
- Reply-bait CTAs work ("what am I missing?", "agree?") but use sparingly — algorithm pattern-matches
- Quote-tweet CTAs ("QT with your version") work for engagement campaigns

**Algorithm signals** (post-Twitter-acquisition X):
- Replies > likes > retweets > views
- Author follower verification status affects reach
- Link-in-post suppresses reach (link-in-reply is common workaround)
- Long-form X (Premium subscriber feature) extends limit to 25K chars but rarely outperforms threads

**Sub-form variations**:
- **Single tweet**: above
- **Thread**: above — 3–7 posts, hook tweet stands alone, link in final post
- **Quote-tweet with commentary**: brand voice using QT to add a take
- **X Long-form**: 25K chars; behaves more like a blog post; deprioritise vs threads unless brand has Premium reach
- **X Spaces**: live audio; not in scope for social copy

### Instagram

**Audience**: consumer-heavy, B2C, lifestyle, creator economy, increasingly B2B for visual brands.

**Reader contract**: visual-first. The image / carousel / Reel does the work; the caption supports.

**Length**: caption 2,200 max; first 125 chars critical (above "more" cutoff).

**Hook rules**:
- Hook in first 125 chars of caption
- For carousels: cover slide hook does the work; caption restates / adds context
- For Reels: hook spoken + on-screen text in first 3 seconds (see TikTok section — same rules)

**Format**:
- **Single image post**: caption with hook → body → CTA. Hashtags in first comment OR end of caption.
- **Carousel**: 7–10 slides (10 is the max IG accepts). Cover slide hook → interior slides build → final slide CTA. Each slide is its own composition.
- **Reels** (vertical video): see [TikTok / Reels / Shorts](#tiktok--reels--shorts) section.
- **Story** (24-hour ephemeral): hook in first 3 seconds; use stickers / polls / questions for engagement; series can chain stories into a "highlight."

**Hashtags**: 5–15 in first comment (preferred — keeps the caption clean) OR end of caption. Mix campaign hashtag + broad topic hashtags + niche/community hashtags. Trending hashtags work if genuinely relevant; forced trending-hashtag insertions backfire.

**CTA**:
- "Save this" + "Send to a friend" drive saves (a strong ranking signal)
- Link in bio reference for traffic ("link in bio" / "tap bio to read")
- "Comment X for the link" via DM-automation tools works but flag as platform-grey
- Story polls / questions for community engagement

**Algorithm signals**:
- Saves + shares > comments > likes (saves rank highest — they're "this is valuable, I want to come back")
- Watch-time on Reels weighs heavily
- Carousel swipe-through rate matters
- Reach is capped per-post; growth comes from a high-ratio post earning algorithmic boost

**Sub-form variations** (covered above): single image / carousel / Reel / story.

### TikTok / Reels / Shorts

**Audience**: TikTok skews younger (Gen Z primary, millennial secondary) but B2B TikTok is real. Reels = IG audience. Shorts = YouTube audience (cross-generational, mobile-pulled).

**Reader contract**: muted scroll. Sound is off by default for ~50% of feed views. Video has 1.5–3 seconds to hook before scroll.

**Length**: 15–90 seconds; 30–45 seconds is the sweet spot for B2B and education content; consumer can go shorter (15–30s).

**Hook rules**:
- **Three signals firing in first 3 seconds**: spoken hook + on-screen text + visual pattern-interrupt
- Without sound: the on-screen text must carry the hook (assume muted)
- Without visuals: the spoken hook must carry the hook (assume eyes-elsewhere)
- Pattern-interrupt visual: high contrast, movement, unexpected composition, face shot, hand gesture — anything that stops a thumb

**Format** (script structure):
- **0–3s Hook**: pattern-interrupt visual + spoken line + on-screen text
- **3–N s Payload**: the substance — story / framework / demo / proof
- **Last 2–3s Close**: verbal CTA + on-screen text echo + final visual beat

**Caption**:
- ≤150 chars typical
- Echo the video's hook in text
- 3–5 hashtags (mix campaign + trending + niche)

**Cover frame** (thumbnail):
- High-contrast moment with on-screen text from the hook
- Faces outperform abstract visuals
- Bright colours outperform muted (in feed competition)

**On-screen text discipline**:
- Short (3–6 words per line) — text on screen is read faster than spoken
- Sans-serif, bold weight, high contrast against background
- Word-by-word reveal (captions style) on TikTok native; sentence-blocks acceptable
- Don't put critical info ONLY in spoken — assume muted

**CTA**:
- Verbal CTA in last 2 seconds: "Follow for more / save this / share with [audience]"
- Caption echo of CTA
- Link in bio reference for traffic
- Comment-prompt CTAs for engagement

**Algorithm signals**:
- Completion rate (watch-through %) is king — videos watched to end get pushed
- Rewatches signal high-value content
- Comments + shares > likes > follows
- Sound-on engagement signals slightly above sound-off (but sound-off is the default)
- Trending sounds boost reach if used early in the trend cycle

**Platform differences within this family**:
- **TikTok**: trend-native, sound-led, edit-heavy. Trending sounds matter most here.
- **Reels (IG)**: same form, less edit-heavy, more brand-polished tolerance, sound-trends less critical
- **Shorts (YouTube)**: better for tutorial / education content; YouTube long-tail discoverability means hooks need to match search intent; less algorithm-trend-dependent

### Facebook

**Audience**: older skewing (35+); community group active; consumer brands still strong; B2B underperforms.

**Reader contract**: feed dominated by family / friends / community groups. Brand posts compete in a softer attention environment but rarely break out.

**Length**: 40–80 words optimal. Longer storytelling allowed if it earns the read.

**Hook rules**:
- Question-led hooks perform well ("Did you grow up [doing X]?")
- Story-first openers work
- Avoid: "[Brand name] is feeling [emoji] at [location]" boilerplate

**Format**:
- Conversational tone
- Line breaks for readability
- Storytelling > marketing
- Photos and videos perform; text-only posts underperform unless the writer is exceptional

**Hashtags**: 0–2. Facebook hashtags rarely move engagement; mostly cosmetic.

**CTA**:
- Comment-prompting questions ("what's your X?") work — algorithm favours comments
- "Tag a friend who..." works for organic spread

**Algorithm signals**:
- Comments > shares > reactions > clicks
- Algorithm strongly favours posts with comment threads (especially threaded replies)
- Outbound links suppress reach
- Group posts can outperform Page posts dramatically

### Threads

**Audience**: Meta's X-alternative; conversational; less news-y; less politicised than X.

**Reader contract**: similar to X but less corporate-formal; more reply-chain than thread-numbered.

**Length**: 500 chars per post.

**Hook rules**:
- Casual + conversational; the corporate-LinkedIn tone fails here
- Less specific-number / contrarian than X; more story-fragment / observation

**Format**:
- Short paragraphs
- Reply-chain rather than numbered thread typically
- "1/" "2/" numbering is rare and slightly out-of-register

**Hashtags**: 0–2.

**CTA**:
- Reply-bait > link
- Story prompts work

**Algorithm signals** (early-platform, evolving):
- Replies + reposts matter
- Cross-posting to Instagram is supported and can extend reach
- Threads still smaller than X — distribution ceiling lower

### Reddit

**Audience**: subreddit-specific; community-native; intolerant of marketing-coded copy.

**Reader contract**: high-signal, low-marketing-tolerance community. Posts that smell of brand or self-promotion get downvoted or removed.

**Length**: variable by subreddit. Read 10 top posts in the target sub before writing.

**Hook rules**:
- Subreddit-native — match the house style of the specific subreddit
- *Never* lead with brand name
- Question-led posts work in advice subs; story-fragments work in narrative subs; data posts work in data subs

**Format**:
- Markdown supported (headers, lists, italics, links)
- Often no formatting — match the subreddit
- Link posts vs text posts: subreddit preference varies; check sidebar rules

**Hashtags**: not used on Reddit.

**CTA**:
- Almost never — Reddit punishes promotional CTAs harder than any major platform
- If brand association is essential, disclose it in the post explicitly ("disclosure: I work for X — this is my own observation")
- AMAs are the formalised "brand on Reddit" pattern with their own conventions

**Algorithm signals**:
- Upvote ratio (% upvoted) matters more than total upvotes
- Comments + discussion depth weighs heavily
- Karma / age of account affects throttling
- Subreddit-specific rules can ban content categories outright — read the sidebar

### Hacker News

**Audience**: software engineers, founders, technologists, VCs.

**Reader contract**: technical-substance-only. Marketing-coded copy is punished even more than Reddit.

**Length**: title ≤80 chars; submission text optional and short.

**Hook rules**:
- Specific + technical title
- Understated > performative
- No marketing words ("revolutionary," "game-changing," "powerful" — all flag)
- Show, don't tell ("Show HN: I built a..." prefix has its own discoverability)

**Format**:
- Title + URL submission is the dominant pattern
- Optional submission text: 1–2 sentences of context
- Comments thread is where the real engagement happens; respond technically and substantively

**Hashtags**: not used.

**CTA**:
- Almost never — same reasoning as Reddit, more so

**Algorithm signals**:
- Upvotes in first 30 mins determine front-page chance
- Title quality matters disproportionately (title is what gets clicked)
- Account history affects throttling
- Self-posts (text submissions) deprioritised vs link posts unless "Ask HN" / "Show HN" prefix

### YouTube

**Long-form** (>3 min): handled by Motion Designer / Video Storyteller. Social/Community Manager doesn't own long-form video copy beyond strategic input.

**Shorts** (<60s): see [TikTok / Reels / Shorts](#tiktok--reels--shorts).

**Title craft for long-form**: specificity + curiosity-loop + keyword (this is search-driven, unlike TikTok). Outliers' "How I [specific result]" / "Why [contrarian claim]" patterns work.

**Thumbnail craft**: face + expression + bold text overlay. Tested A/B. Static Designer territory.

## Hashtag taxonomy principles

When the Social/Community Manager authors a campaign hashtag taxonomy in Workflow 1, follow this structure.

### Campaign hashtag

- **One** (occasionally two if a B2B + B2C split)
- Short, memorable, **ownable** (search to confirm not in use, hijacked, or deprecated)
- CamelCase for accessibility: #ReadableHashtag not #readablehashtag (screen readers parse CamelCase by syllable boundary)
- Tested across platforms — same tag works on LinkedIn / X / IG / TikTok / Threads

### Topic hashtags

- 5–10 evergreen hashtags the campaign sits inside
- Mix:
  - **High-volume / broad** (1–2): reaches widely, harder to rank in
  - **Mid-volume / topic-specific** (3–5): better reach-to-engagement ratio; sweet spot
  - **Niche / community** (1–3): smaller reach, higher engagement, signals tribal membership

### Platform-specific placement

| Platform | Count | Placement |
|---|---|---|
| LinkedIn | 3–5 | End of post |
| X | 0–2 | Inline |
| Instagram | 5–15 | First comment (preferred) OR end of caption |
| TikTok | 3–5 | Caption |
| Threads | 0–2 | Inline |
| Facebook | 0–2 | Inline (rarely move engagement) |
| Pinterest | 2–5 | End of description (Pinterest is search-driven; hashtags less important than keyword stuffing) |

### Banned / avoid list

- Competitor-owned hashtags (don't feed their algorithm)
- Hijacked-by-bad-actors hashtags (do search before locking)
- Deprecated hashtags from previous campaigns (don't reuse old campaign tags unless intentional)
- Overly broad hashtags (#marketing #business) — high volume, near-zero signal
- Politically-charged hashtags unless campaign is explicitly political

## Accessibility checklist

Every social asset has accessibility requirements that the Social/Community Manager flags (writes the alt text / caption plan; Designer or operator executes).

**Images** (every post with an image):
- Alt text written, descriptive (not "image" or "graphic")
- Alt text describes the image's *function* in the post, not just its appearance
- Decorative images can use alt="" but flag the choice
- Text-on-image: critical info also in caption (alt text alone is not enough for sighted screen reader users in noisy environments)

**Video** (every post with video):
- Captions / subtitles burnt in OR provided as a track
- Captions accurate (not auto-generated and uncorrected)
- Captions readable on mobile (sans-serif, bold, high contrast)
- No critical info conveyed only by sound

**Text formatting**:
- No all-caps for more than 1 word (screen readers spell each letter)
- No emoji-only sentences (screen reader reads emoji name aloud — "smiling face with smiling eyes")
- Symbols and special characters used sparingly (∴ ✓ etc. — screen readers read them)
- Line breaks used for readability, not Unicode tricks for "fake bold" — that breaks screen readers

**Hashtag formatting**:
- CamelCase multi-word hashtags

**Colour and contrast** (visual posts):
- Text-on-image meets WCAG AA contrast ratio (4.5:1 for normal, 3:1 for large)
- Critical info not conveyed by colour alone (red/green callouts also use shape or label)

## Disclosure mandatories

Flag in §10 Production notes if any apply but are not yet present in the draft.

**FTC (US) endorsement guides**:
- Paid partnership: "Paid partnership with X" / #ad / #sponsored — required in conspicuous placement
- Influencer / creator partnerships: #ad, not buried in hashtag stack
- Free product / affiliate: disclose materially
- Employee posting about employer: disclose employment relationship

**Platform-specific disclosure tools**:
- Instagram + Facebook: "Paid partnership" tag (preferred over hashtag)
- TikTok: "Branded content" disclosure setting
- LinkedIn: "Paid partnership" tag where available; #ad otherwise

**Regulated industries**:
- Financial services: "Not financial advice" / regulator-specific disclosures (FCA UK / SEC US / ASIC AU)
- Healthcare / pharma: FDA disclosure rules; "consult your doctor" language; off-label promotion banned
- Crypto: risk disclosures required in some jurisdictions
- Alcohol / gambling: age-gating, regional restrictions

**Brand-mandated tags** (per Brief §9):
- Trademark notations (™ ®) per legal team's rules
- Attribution credits (photographer / illustrator / data source)
- Specific call-outs ("In partnership with [X]")

## Community-response classification taxonomy

When a live comment / DM / mention surfaces, the Social/Community Manager classifies before responding. The classification determines the response approach.

| Type | Definition | Default action |
|---|---|---|
| **Agree** | Comment supports the post's point, adds context, extends the conversation | Acknowledge + extend (don't just say "thanks!") — surface a follow-up thought |
| **Disagree (good faith)** | Comment respectfully challenges the post's claim with substance | Engage *if* the disagreement is in good faith and sharpens the conversation; otherwise let the post stand |
| **Disagree (bad faith)** | Comment attacks the post's framing without engaging the substance | Do not respond OR a single firm reply that names the bad faith; do not engage repeatedly |
| **Question** | Comment asks a genuine question | Answer directly. If operational → route to support and tag the operator. If competitive → route to Sales / Brand. If legal-adjacent → route to Legal. |
| **Praise** | Comment expresses appreciation without substance | Thank + surface a follow-up (don't waste the engagement on "thanks!") |
| **Complaint** | Comment expresses dissatisfaction with the brand / product | Acknowledge + route to support; never argue publicly; offer to take it to DM |
| **Operational issue** | Specific service / product / billing / access problem | Do not resolve publicly; route to support; respond publicly only to acknowledge and direct |
| **Competitive comparison / pricing claim** | Comment compares the brand to a competitor or makes a specific pricing / feature claim | Route to Sales / Brand; respond cautiously with verified info or hold |
| **Legal-adjacent question** | Comment asks about a claim that has legal exposure (regulated industry, IP, contract terms) | Route to Legal; respond only with Legal-cleared language; default hold |
| **Crisis-tone / virality risk** | Comment is part of a coordinated attack, going viral, or could escalate | Escalate to Brand + user (operator + CMO); write a holding reply; do not improvise |
| **Spam / bot** | Generic promotional reply, obvious bot, off-topic spam | Do not respond; flag for operator to remove |
| **Troll** | Bait, harassment, off-topic provocation | Do not respond; flag for operator to assess (delete vs ignore vs ban); silence is usually correct |

## Posting-time defaults

Defaults the Social/Community Manager uses for posting cadence in Workflow 1 unless the Brief / Brand provides better signal. State assumptions explicitly; the operator can override.

| Platform | Audience timezone consideration | Default time slots |
|---|---|---|
| LinkedIn (B2B) | Posting-audience timezone (where most readers are) | Tue–Thu 7–9am OR 12–1pm |
| LinkedIn (general) | Same | Tue–Thu 7–9am; Sun 5–7pm catches Monday-prep readers |
| X | Spread across day; news cycle matters | 9–11am + 2–4pm + 7–9pm windows |
| Instagram (consumer) | Audience leisure time | Lunch (12–1pm) + evening (6–9pm); weekend strong |
| Instagram (B2B) | Audience work-adjacent | 11am–1pm weekdays |
| TikTok | Evening + weekend | 6–10pm; Fri–Sun strong |
| Facebook | Older audience pattern | Early morning (7–9am) + evening (7–9pm); midweek slightly stronger |
| Threads | Same as X but slightly later | 10am–12pm + 8–10pm |
| Reddit | Subreddit-specific; check subreddit traffic patterns | n/a — varies wildly |
| YouTube Shorts | Evening | 6–10pm |

**Campaign-phase ramp**:
- **Phase 1 (build)**: lower frequency; consistent presence; testing what resonates
- **Phase 2 (launch)**: peak frequency; multi-touch per platform per day acceptable; coordinated cross-platform moments
- **Phase 3 (sustain)**: dropped frequency, higher quality bar, post-launch follow-up + community engagement
