Deployment guide

Get your own copy running on your computer. No technical experience needed. Three quick installs, then the system sets itself up — about 30–45 minutes, most of it waiting on downloads.

Don't want to do the setup yourself? Whoever gave you this system can do steps 1–4 for you on a screen-share in about half an hour. You only ever need the chat (step 5 onward).
Step 1 — Install Python

This runs the system's behind-the-scenes scripts.

  1. Go to python.org/downloads and click the big yellow Download Python button.
  2. Open the file you just downloaded.
  3. IMPORTANT: on the first screen, tick "Add Python to PATH" (bottom of the window) before clicking Install Now. This one tick prevents the most common setup problem.
  4. Let it finish, then click Close.
Step 2 — Install Claude Code

This is the app you'll actually use.

  • Follow Anthropic's official installer: code.claude.com/docs/en/setup (Windows).
  • New to Claude Code? The 5-minute quickstart: code.claude.com/docs/en/quickstart.
Step 3 — Get the system's files

You'll have received the system one of two ways — follow whichever matches:

A — You were sent a folder or a ZIP file. If it's a ZIP, right-click → Extract All. Move the folder into your OneDrive/Dropbox folder (keeps it backed up) and rename it Marketing AI System.

B — Download it from GitHub (no GitHub account needed): open github.com/abraun64/soundtrak-ai-marketing-system, click the green < > Code button → Download ZIP, then extract and move it as in A.

Either way you end up with one folder called Marketing AI System full of files — that folder is your system.

Step 4 — Let the system set itself up

Open Claude Code and open the folder from Step 3 as your project. In the chat box, type:

set yourself up

It installs the last few pieces it needs, checks everything works, and turns on automatic backup. If anything's missing it tells you the one thing to fix in plain language. When it says READY, you're done with setup.

Step 5 — Start using it

In the chat, type "Onboard <your business name>". The system interviews you about your business — voice, look, where you publish — and builds your foundation (required before any campaign can start). Then say "start a campaign". Full walk-through: Getting started.

If the health check shows red

Step 4 runs a "doctor" that checks your setup. A red [FAIL] line always prints the exact fix. The usual culprits:

Red lineWhat it meansFix
python module: … MISSINGa helper library didn't installtype "set yourself up" again — it retries
playwright: chromium not installedthe preview browser didn't downloadsame — say "set yourself up" again
"python is not recognised"Python installed without "Add to PATH"re-run the Step 1 installer and tick "Add Python to PATH"

Do I need a GitHub account?

No — not to download the system, and not to use it day to day. GitHub shows up in two separate ways, and only the optional one needs an account of your own:

GitHub touchpointWhose it isAccount needed?
The public page you download from (Step 3B)the people who make the systemNo — anyone can download
An optional online backup of your own workyours (a free private repo)Yes — but only if you want it

Backup & upgrades (the short version)

  • Backup is automatic once set up (Step 4): your work saves at the end of every session. The safety net is that local save plus your OneDrive/Dropbox folder — an offsite copy with zero extra setup. That's enough for most people, no GitHub account required.
  • Want a separate online backup in your own account? Optional — create a free private repo, then say "set yourself up" again and the system connects it.
  • Upgrades: your copy is yours and keeps working as-is — you never have to upgrade. If a newer version is released, whoever gave you the system can hand you the update.
